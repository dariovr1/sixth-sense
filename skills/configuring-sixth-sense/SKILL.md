# Setup Sixth Sense

Run this skill once per project to configure how Sixth Sense dispatches subagents.

## Interview

Ask the user the following questions, one at a time. For each question, provide a recommended answer:

Note: Opus is never used for task routing or implementation dispatch — there is no `opus-low/medium/high` in the Effort Routing table, and none will be added. The only way Opus runs is the narrow second-opinion pass in Q4 below, on by default per benchmark evidence (see Q4) but disable-able.

### 1. Authorized models
> Which models can subagents use?
> - Sonnet only (no Haiku batching — everything, including mechanical tasks, runs on Sonnet)
> - Sonnet + Haiku (recommended — trivial, well-specified tasks batch onto Haiku for a fraction of the cost; see Q3)

### 2. Default effort for Sonnet
> What effort level should Sonnet use for standard work?
> - **low** — well-defined tasks, low ambiguity risk
> - **medium** (recommended) — good tradeoff for everyday work
> - **high** — complex tasks, architectural decisions

### 3. Haiku batching
> Trivial tasks (single-file change, <20 lines, no architectural decision) are batched onto Haiku **by default** — the eligibility bar is already narrow enough to act as a guardrail. Turn this off only if you want every task on Sonnet regardless of cost.
> - **Keep it on** (default, recommended)
> - **No** — all tasks go to Sonnet

### 4. Opus second-opinion review
> Beyond the normal review, `synthesizing-spec` and `requesting-code-review` dispatch one extra, read-only Opus pass by default — not to redo the review, but to catch what a per-file/per-hunk pass structurally can't: whole-design architectural concerns, "there's a library for this," cross-cutting failure modes. It never touches routine dispatch or implementation, only this one bounded checkpoint, and only when the prior review already exists for it to build on. Benchmarked at ~4% of a full review pipeline's cost for a real bug class Sonnet's hunter missed — on by default on that basis.
> - **Yes** (default, recommended) — expect it to run rarely (once per finished spec, once per finished review), not per task
> - **No** — skip it, rely on the standard review only (e.g. tight budget constraints)

## Output

After the answers, generate the `.sixth-sense/model-preferences.md` file in the current project:

```markdown
---
models:
  sonnet: true
  haiku: {true/false}
default-effort: {low/medium/high}
haiku-batching: {true/false}
opus-review: {true/false, default true}
max-parallel-agents: 4
---

# Notes
Generated by /setup-sixth-sense on {date}.
To reconfigure, run /setup-sixth-sense again.
```

Confirm to the user that the file has been created and that the preferences will be read automatically by `dispatching-parallel-agents` on every dispatch.
